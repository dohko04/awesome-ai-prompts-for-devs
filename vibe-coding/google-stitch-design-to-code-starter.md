## Why This Matters

Google Stitch 2.0 isn't just a design tool — it's a **design-to-code pipeline**. The new MCP server lets your coding agent pull UI specs directly from Stitch, and the SDK exports real React components. Non-designers can now go from idea → working app in minutes, not days.

## Quick Decision: When to Use Stitch

```yaml
# stitch-decision.yaml — Is Stitch right for your workflow?

use_stitch:
  - "Prototyping UI before coding (voice or text prompts)"
  - "Generating React components from visual designs"
  - "Non-designers needing production-quality mockups"
  - "Teams wanting design → code handoff via MCP"
  - "Rapid iteration with infinite canvas + voice mode"

skip_stitch:
  - "You already have a mature Figma workflow with dev handoff"
  - "Your app is purely backend/API with no UI"
  - "You need pixel-perfect brand compliance (Stitch is AI-generated)"
```

## Stitch 2.0 → Coding Agent Workflow

```markdown
## Step 1: Design in Stitch
1. Open stitch.google.com
2. Describe your UI via text or voice: "Dashboard with user metrics, dark theme, sidebar nav"
3. Use infinite canvas to iterate on multiple screens
4. Manual edit: click any element to tweak text, spacing, colors

## Step 2: Export via MCP or SDK
Option A — MCP Server (for Cursor, Claude Code, Antigravity):
  1. Enable Stitch MCP server in project settings
  2. In your coding agent, connect to Stitch MCP endpoint
  3. Agent pulls design specs, component tree, and styles automatically

Option B — SDK Export (for AI Studio, standalone):
  1. Select screens → Export → React Components
  2. Download as zip or push to AI Studio
  3. Components include Tailwind/CSS modules + responsive breakpoints

## Step 3: Refine in Code
- Use DESIGN.md (auto-generated by Stitch) as your source of truth
- Coding agent reads DESIGN.md for layout decisions
- Iterate: change design in Stitch → re-export → diff in code
```

## DESIGN.md Template (Stitch Auto-Generated)

```markdown
# DESIGN.md — Generated by Google Stitch 2.0

## Design System
- **Theme:** Dark
- **Primary Color:** #6366F1
- **Font:** Inter, system-ui
- **Border Radius:** 8px
- **Spacing Scale:** 4px base

## Screens
### Dashboard (/)
- Sidebar: 240px fixed, collapsible
- Main: 3-column metric cards + chart area
- Header: user avatar + search + notifications

### Settings (/settings)
- Tab layout: Profile | Security | Billing
- Form components: inputs, toggles, selects

## Components Exported
- `<MetricCard />` — props: title, value, trend, icon
- `<Sidebar />` — props: items[], collapsed
- `<DataChart />` — props: data[], type (line|bar|area)

## MCP Integration
- Endpoint: stitch-mcp://project-{id}
- Available tools: get_design_spec, get_component_tree, get_styles
```
