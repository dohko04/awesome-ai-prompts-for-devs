## Quick Migration Map

| Browser Agent Use Case | Coding Agent Alternative |
|----------------------|------------------------|
| Web scraping | RAG pipeline + API integration |
| Form filling | API-first automation + MCP tools |
| UI testing | Keep browser (still valid) |
| Data extraction | LLM structured output + API calls |
| Workflow automation | OpenClaw/Cursor Automations |
| Content generation | Direct LLM generation |

## 5 Winning Patterns (Post-Browser Era)

1. **OpenClaw Autonomous Agent** — Reads specs, writes code, runs tests, commits
2. **Cursor Automations** — Event-driven AI triggered by git events
3. **MCP Tool Mesh** — API-first data access replacing browser scraping
4. **Multi-Agent Team** — Architect + Coder + Reviewer + Tester
5. **RAG Knowledge Agent** — Replaces browser-based knowledge scraping

## Quick Decision Framework

```
Is your agent scraping websites?
  → YES: Replace with API calls + MCP servers
  → NO: Is it automating browser-based workflows?
    → YES: Replace with OpenClaw/Cursor coding agent
    → NO: Is it doing UI testing?
      → YES: Keep browser agent (still the best approach)
      → NO: Evaluate coding agent patterns above
```
