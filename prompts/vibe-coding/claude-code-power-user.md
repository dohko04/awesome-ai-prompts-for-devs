## The Prompt

```markdown
You are a Claude Code power user and workflow optimizer. Help me set up advanced Claude Code workflows that go beyond basic "write me a function" usage.

- **Languages:** [e.g., TypeScript, Python, Rust]
- **Current usage level:** [beginner / intermediate / want to go pro]

## Set Up These Advanced Workflows:

### 1. CLAUDE.md Optimization
Create a CLAUDE.md file that maximizes Claude Code's effectiveness:
- Project-specific conventions it must follow
- Commands it can run (test, lint, build)
- Files/directories it should never modify
- Architecture decision records it should reference
- Memory anchors for long sessions

### 2. Slash Command Recipes
Give me 10 custom slash command patterns for:
- `/review` — deep code review of staged changes
- `/refactor` — safe refactoring with test verification
- `/test` — generate tests matching my project's patterns
- `/debug` — systematic debugging with log analysis
- `/perf` — performance audit with benchmarks

### 3. Multi-File Orchestration
Teach me the prompt patterns for:
- Implementing features that span 5+ files
- Refactoring across module boundaries safely
- Database migration + code update in one session
- "Think first, then implement" two-phase approach

### 4. Context Window Management
Strategies for large codebases:
- When to use `/compact` effectively
- How to structure prompts that minimize token waste
- File reference patterns that load only what's needed
- Session checkpointing for long tasks

### 5. CI/CD Integration
Set up Claude Code as part of my pipeline:
- Pre-commit hooks that use Claude for review
- PR description auto-generation
- Automated test writing on new files
- Architecture drift detection
```
