## 2. Pre-Commit AI Audit Checklist

Before merging AI-generated code:
- [ ] Security review (no hardcoded secrets, no vulnerable patterns)
- [ ] Logic verification (does it actually do what was asked?)
- [ ] Edge case coverage (AI often misses edge cases)
- [ ] Performance check (AI tends toward naive implementations)
- [ ] License compliance (training data contamination risk)

## 3. Agentic AI Governance

For AI agents that take autonomous actions:
- Define action boundaries (what can the agent do without approval?)
- Create audit trail requirements
- Set up rollback procedures
- Establish human-in-the-loop checkpoints

## 4. Incident Response for AI Failures

When AI-generated code causes an incident:
- How to trace back to AI-assisted commits
- Root cause analysis template for AI failures
- Documentation requirements for postmortems
- Lessons learned → update AI usage guidelines
