## The Prompt

```markdown
You are a Physical AI architecture consultant. I'm exploring building a Physical AI application and need help selecting the right stack.

## My Project
- **Domain:** [manufacturing / logistics / autonomous vehicles / humanoid robotics / drones / warehouse automation / other: ___]
- **Current state:** [concept / prototype / simulation-only / real-world testing]
- **Budget:** [hobby / startup / enterprise]
- **Hardware:** [consumer GPU / cloud GPU / no GPU yet]

## Help me with:

### 1. Simulation Platform Selection
Compare these for my use case (pick top 2):
- NVIDIA Omniverse / Isaac Sim
- Gazebo + ROS 2
- MuJoCo
- Unity ML-Agents
- Ai2 MolmoSpaces (open-source)

For each, tell me: best for, cost, learning curve (weeks), and whether it fits my project.

### 2. Foundation Model Selection
Recommend models for:
- **Perception** (what the robot sees)
- **Planning** (what the robot decides)
- **Control** (how the robot moves)

### 3. Sim-to-Real Reality Check
Give me an honest assessment:
- Expected transfer success rate for my domain
- Top 3 sim-real gap challenges I'll face
- Minimum viable testing protocol before deploying on real hardware

### 4. First 30-Day Plan
Week-by-week plan to go from zero to a working simulation prototype.

Keep it practical. I want to build, not just read papers.
```
