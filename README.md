## 🐛 Debugging

### The Bug Detective
```
I have a bug. Here's what I know:

Expected behavior: [what should happen]
Actual behavior: [what happens instead]  
Reproduction steps: [how to trigger it]

Code:
[relevant code]

Error output (if any):
[errors/logs]

Help me:
1. Identify the most likely root cause
2. Explain WHY it's happening
3. Provide the fix
4. Suggest how to prevent similar bugs
```

### The Performance Profiler
```
This code is slow. Profile it:

[paste code]

1. Identify the O(n) complexity of each function
2. Find the bottleneck
3. Suggest optimizations ranked by impact
4. Show the optimized version with before/after complexity
```

## ⚡ Code Generation

### The Architecture Kickstarter
```
I need to build [describe feature/system].

Tech stack: [your stack]
Scale: [expected users/data volume]
Constraints: [time, budget, existing systems]

Give me:
1. High-level architecture diagram (ASCII)
2. Key components and their responsibilities
3. Data flow between components
4. The top 3 technical risks and mitigations
5. A phased implementation plan
```

### The Test Writer
```
Write comprehensive tests for this code:

[paste code]

Requirements:
- Use [testing framework]
- Cover happy path, edge cases, and error cases
- Include integration tests where appropriate
- Mock external dependencies
- Each test should have a clear, descriptive name
```

## 🗃️ Database & SQL

### The Query Optimizer
```
Optimize this SQL query:

[paste query]

Table schemas:
[paste relevant schemas]

Current execution time: [if known]
Database: [PostgreSQL/MySQL/etc]

Provide:
1. Explain what the query does in plain English
2. Identify performance issues
3. Suggest indexes
4. Rewrite the optimized query
5. Explain why each change helps
```

## 🔌 API Design

### The API Designer
```
Design a REST API for [feature/resource].

Context: [what the API does]
Users: [who consumes it]
Auth: [authentication method]

Provide:
1. Endpoint list with methods and paths
2. Request/response schemas (JSON)
3. Error response format
4. Pagination strategy
5. Rate limiting recommendations
```
